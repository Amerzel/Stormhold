# AGENTS.md — Stormhold Game Implementation

Godot 4.x action RPG vertical slice. All game data comes from the Forge tool pipeline — **nothing should be hardcoded**. Read entity stats, encounters, quests, and combat rules from the JSON files in zone_packs/.

## Quick Start

1. Open this project in Godot 4.4+
2. All game data is pre-loaded in zone_packs/stormhold-clearing/
3. The GameData autoload reads it at runtime
4. Implement scenes and scripts that consume this data

## Architecture

zone_packs/ contains read-only data from the Forge pipeline:
- entities/entity_catalog.json — 23 entities (enemies, weapons, abilities, etc.)
- encounters/encounters.json — 6 spawn placements with tile positions
- quests/quests.json — Quest graph (kill goblins → troll → shaman)
- game_rules.json — Damage formulas, stat curves, enemy tiers
- terrain/resolved.v1.json — 80×60 tile map
- zone.v1.json — Zone regions, spawn point

scripts/managers/ contains three autoloads (already implemented):
- game_data.gd — Loads and serves all pack data
- combat_manager.gd — Damage calculations from ForgeRules
- quest_manager.gd — Quest state machine from ForgeQuest graphs

## Key Conventions

- **Data-driven:** Every numeric value (HP, damage, speed, cooldowns) must come from GameData, never hardcoded
- **Units:** All distances are in tiles (1 tile = 32px). Speeds are tiles/second. Times are seconds.
- **Entity IDs:** Use the entity id field as the key (e.g., "enm-goblin-grunt", "wpn-iron-sword")
- **Pixel-perfect:** nearest neighbor filtering (no smoothing)

## How to Access Game Data

```gdscript
# Get any entity by ID
var grunt = GameData.get_entity("enm-goblin-grunt")

# Get player class definition
var warrior = GameData.player_class

# Calculate damage
var dmg = CombatManager.calculate_physical_damage(12.0, 1.0, 6.0)

# Roll loot
var loot_table = GameData.get_loot_table("loot-goblin-common")
var drops = CombatManager.roll_loot(loot_table)

# Quest tracking
QuestManager.start_quest("quest:clear-the-clearing")
QuestManager.report_kill("enm-goblin-grunt")
```

## Anti-Patterns

- Do not hardcode stats — read from GameData
- Do not edit zone_packs/ — generated by Forge pipeline
- Do not duplicate combat formulas — use CombatManager
- Do not create entity types not in the catalog
